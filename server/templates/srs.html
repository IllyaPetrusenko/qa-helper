В этом блоке мы научимся с Вами тестировать пользовательский интерфейс.

Немного информации:

Представим себе, что программисты нам отдали в тестирование всего две страницы. Страницу создания "Статьи расходов"
и страницу создания "Источника финансирования". Теперь нам необходимо это протестировать, имея входящую информацию о этих сущностях.

Требования:

BR-1 У пользователя из интерфейса должна быть возможность перейти на страницу создания "Статьи расходов"

BR-2 Пользователь должен заполнить следующие обязательные поля:

BR-2-1 Title

BR-2-2 Tender classification

BR-2-3 Planning period start date

BR-2-4 Planning period end date

BR-2-5 Buyers name

BR-2-6 Buyers identifier

BR-2-7 Buyers scheme

BR-2-8 Buyers Legal name

BR-2-9 Buyers Street address

BR-2-10 Buyers Postal code

BR-2-11 Buyers country

BR-2-12 Buyers region

BR-2-13 Buyers locality

BR-2-14 Contact point name

BR-2-15 Contact point email

BR-2-16 Contact point phone

VR-1 Валидационные правила:

VR-1-1 Planning period start date должно быть меньше Planning period end date

VR-1-2 Planning period end date должно быть больше Planning period start date

VR-1-3 Поле Buyers scheme может принимать значение только "MD-IDNO"

VR-1-3 Поле Buyers country может принимать значение "Moldova"
